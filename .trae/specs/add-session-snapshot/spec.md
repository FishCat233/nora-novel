# 会话快照保存和加载系统 Spec

## Why

用户需要能够手动保存当前的对话会话状态（存档），并在需要时恢复到之前的状态（读档）。配合自动保存最近10条对话记录，可以实现重新生成内容、回溯对话历史等功能，提升小说创作的灵活性和可控性。

## What Changes

* 新增 `SnapshotStorage` 存储类，用于保存/加载会话快照

* 在侧边栏添加"存档/读档"管理 UI，支持用户手动操作

* 实现自动保存机制，保留最近10条对话记录用于重新生成

* 快照数据包含：消息历史、当前模块ID、时间戳、用户自定义名称

## Impact

* 新增存储模块 `storage/snapshot.py`

* 修改 `sidebar.py` 添加存档管理入口（用户手动操作）

* 修改 `chat_input.py` 添加自动保存逻辑

* 修改 `main.py` 初始化快照存储和处理读档逻辑

## ADDED Requirements

### Requirement: 手动存档功能

系统 SHALL 允许用户手动保存当前会话状态为存档点。

#### Scenario: 用户手动存档

* **GIVEN** 用户在聊天页面

* **WHEN** 用户在侧边栏输入存档名称并点击"保存存档"

* **THEN** 系统 SHALL 将当前会话状态保存为 JSON 文件

* **AND** 文件命名格式为 `{timestamp}_{name}.json`

* **AND** 存储路径为 `{SNAPSHOT_PATH}/`

* **AND** 显示保存成功提示

#### Scenario: 查看存档列表

* **GIVEN** 用户在聊天页面

* **WHEN** 查看侧边栏的"存档管理"区域

* **THEN** SHALL 显示所有已保存的存档列表

* **AND** 显示每个存档的名称、保存时间、消息数量

### Requirement: 手动读档功能

系统 SHALL 允许用户加载之前保存的存档点。

#### Scenario: 用户加载存档

* **GIVEN** 用户在存档列表中看到一个存档

* **WHEN** 点击该存档的"加载"按钮

* **THEN** 系统 SHALL 恢复该存档的会话状态

* **AND** 恢复 agent.messages、current\_module\_id

* **AND** 页面刷新后显示恢复的对话历史

#### Scenario: 用户删除存档

* **GIVEN** 用户在存档列表中看到一个存档

* **WHEN** 点击该存档的"删除"按钮

* **THEN** 系统 SHALL 删除该存档文件

* **AND** 更新存档列表显示

### Requirement: 自动保存最近对话

系统 SHALL 自动保存最近10条对话记录，用于重新生成功能。

#### Scenario: 自动保存机制

* **GIVEN** 用户发送消息并收到回复

* **WHEN** 每次对话完成后

* **THEN** 系统 SHALL 自动保存最近10条对话到 `{WIKI_PATH}/../snapshots/.auto_save.json`

* **AND** 只保留用户消息和助手回复（不含系统消息和工具调用）

#### Scenario: 重新生成内容

* **GIVEN** 用户想要重新生成某条回复

* **WHEN** 用户点击"重新生成"按钮（在对应消息旁）

* **THEN** 系统 SHALL 从自动保存的历史中恢复上下文

* **AND** 删除该条回复及之后的消息

* **AND** 重新调用 Agent 生成新回复

### Requirement: 存档数据结构

系统 SHALL 使用统一的 JSON 格式存储存档。

#### Data Structure

```json
{
  "version": "1.0",
  "timestamp": "2026-03-23T17:30:00",
  "name": "用户自定义名称",
  "current_module_id": "common_helper",
  "messages": [
    {
      "role": "user",
      "content": "..."
    },
    {
      "role": "assistant",
      "content": "...",
      "tool_calls": null
    }
  ],
  "message_count": 5
}
```

## UI Design

### 侧边栏存档管理区域

```
📁 存档管理
─────────────
[输入框: 存档名称...]
[💾 保存存档]

已保存的存档:
┌─────────────────────────┐
│ 📄 第一章构思             │
│ 2026-03-23 17:30  5条消息 │
│ [加载] [删除]            │
├─────────────────────────┤
│ 📄 角色设定讨论           │
│ 2026-03-23 16:15  12条消息│
│ [加载] [删除]            │
└─────────────────────────┘
```

### 消息重新生成按钮

* 在每条助手消息的右上角显示"🔄 重新生成"按钮

* 点击后确认是否重新生成

* 重新生成时显示加载状态

## Notes

* 存档存储使用 JSON 格式，便于人工查看和备份

* 自动保存文件 `.auto_save.json` 为隐藏文件，不显示在存档列表中

* 加载存档时会完全替换当前会话状态，未保存的当前对话将丢失

* 考虑添加存档数量限制（如最多50个），防止磁盘空间无限增长

* 重新生成时保留该消息之前的所有上下文

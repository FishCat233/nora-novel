# 自动存档功能 Spec

## Why

用户需要能够在每次对话后自动保存会话状态，以便在需要时回溯历史对话。自动存档功能将在每次对话完成后自动保存快照，最多保留10条自动存档，当第11条存档尝试保存时，会删除最早的自动存档。

## What Changes

* 新增 `AutoArchiveManager` 自动存档管理类，用于管理自动存档的生命周期

* 修改 `sidebar.py`，在默认折叠栏中显示自动存档列表

* 修改 `chat_input.py`，在每次对话完成后触发自动存档

* 修改 `snapshot.py`，添加自动存档专用存储方法

## Impact

* 新增存储模块 `storage/auto_archive.py`（或扩展 `snapshot.py`）

* 修改 `sidebar.py` 添加自动存档显示区域（默认折叠）

* 修改 `chat_input.py` 在对话完成后调用自动存档

* 修改 `main.py` 初始化自动存档管理器

## ADDED Requirements

### Requirement: 自动存档管理

系统 SHALL 在每次对话完成后自动保存会话快照，最多保留10条，采用循环覆盖策略。

#### Scenario: 自动保存对话

* **GIVEN** 用户完成一次对话（发送消息并收到回复）

* **WHEN** 对话流完全结束后

* **THEN** 系统 SHALL 自动保存当前会话状态

* **AND** 自动存档文件命名格式为 `auto_archive_{index}.json`，index 为 0-9

* **AND** 存储路径为 `{SNAPSHOT_PATH}/auto_archives/`

* **AND** 新存档 SHALL 替换最旧的存档（循环覆盖）

#### Scenario: 自动存档数量限制

* **GIVEN** 系统已保存10条自动存档（index 0-9）

* **WHEN** 新的对话完成并触发自动存档

* **THEN** 系统 SHALL 删除 index 为0的最旧存档

* **AND** 将现有存档 index 0-8 依次后移为 1-9

* **AND** 将新存档保存为 index 9（最新）

### Requirement: 侧边栏自动存档显示

系统 SHALL 在侧边栏显示自动存档列表，默认折叠。

#### Scenario: 查看自动存档

* **GIVEN** 用户在聊天页面

* **WHEN** 查看侧边栏的"自动存档"区域

* **THEN** SHALL 显示折叠的"🕐 自动存档"区域

* **AND** 展开后显示所有自动存档列表（按时间倒序）

* **AND** 每个存档显示：序号、保存时间、消息数量

#### Scenario: 加载自动存档

* **GIVEN** 用户在自动存档列表中看到一个存档

* **WHEN** 点击该存档的"加载"按钮

* **THEN** 系统 SHALL 恢复该存档的会话状态

* **AND** 恢复 agent.messages、current_module_id

* **AND** 页面刷新后显示恢复的对话历史

### Requirement: 自动存档数据结构

系统 SHALL 使用统一的 JSON 格式存储自动存档。

#### Data Structure

```json
{
  "version": "1.0",
  "timestamp": "2026-03-23T17:30:00",
  "index": 9,
  "current_module_id": "common_helper",
  "messages": [
    {
      "role": "user",
      "content": "..."
    },
    {
      "role": "assistant",
      "content": "...",
      "tool_calls": null
    }
  ],
  "message_count": 5
}
```

## UI Design

```
📁 存档管理
─────────────

已保存的存档:

▼ 🕐 自动存档 (10)
─────────────────────────
┌─────────────────────────┐
```

### 自动存档特性

* 自动存档区域默认折叠（`expanded=False`）

* 标题显示当前自动存档数量，如"🕐 自动存档 (10)"

* 存档按时间倒序排列，最新的显示在最上面

* 每个存档显示：序号（#1-#10）、时间、消息数、加载按钮

* 自动存档只提供"加载"功能，不提供删除功能（系统自动管理）

## Notes

* 自动存档与用户手动存档分离存储，避免混淆

* 自动存档只保留最近10条，采用循环队列方式管理

* 加载自动存档时同样需要确认对话框，防止误操作

* 自动存档触发时机：在流式输出完全结束且工具调用处理完成后

* 考虑在自动存档中只保留用户和助手消息（排除系统消息和工具消息）以节省空间
